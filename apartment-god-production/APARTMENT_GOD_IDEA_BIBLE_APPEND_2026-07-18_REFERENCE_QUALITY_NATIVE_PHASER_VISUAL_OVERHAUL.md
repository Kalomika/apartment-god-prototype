# Apartment God Reference Quality Native Phaser Visual Overhaul Directive

Date: 2026-07-18
Status: ACTIVE DIRECTIVE
Branch: phaser-migration-2
Reference source: Kam supplied high angle top down anime apartment mockup in ChatGPT

## Locked quality target

Apartment God must be visually rebuilt to the quality level of Kam's supplied apartment mockup, not merely moved closer to it. The reference is a quality, construction, material, character, lighting, and interface target rather than an exact floor plan.

The game must retain its current room footprints, gameplay systems, object purposes, routes, interactions, and true 2D Phaser structure unless a specific physical form is demonstrably incorrect. Correct object forms from the reference, including properly constructed top down kitchen fixtures, refrigerator, sink, sectional couch, dining furniture, bathroom fixtures, stairs, beds, and adult character anatomy, should replace procedural blocks and generic category sprites.

## Visual law

- Native Phaser remains the runtime and renderer.
- Do not restore the old Canvas compatibility renderer or offscreen Canvas texture bridge.
- Use grounded mature anime production art with a high angle top down camera and restrained visible object sides for readability.
- Preserve a flat playable floor plane, true 2D navigation, native Phaser depth, object collision, and mobile readability.
- Furniture and architecture must read as real designed objects, not squares, blocks, icons, labels, bathroom sign figures, blobs, toys, mascots, or category placeholders.
- Characters must match the reference level with adult anatomy, readable clothing, believable hair, shoulders, limbs, and directional construction.
- Character motion remains exactly 8 FPS for the anime timing target.
- Every meaningful activity requires a distinct object aware animation identity. Generic interact, sit, workout, shower, sleep, or use object poses are not acceptable final states.
- Materials must be separated by value, color, light, edge, texture, and cast shadow rather than heavy outlines.
- Lighting should use controlled cool and warm separation without obscuring gameplay readability.
- Objects must remain individually replaceable and interactive. Do not bake all furniture into one untouchable room image.

## Preservation rule

Do not arbitrarily change established forms, footprints, object IDs, routes, click targets, actions, or room purposes. Improve incorrect physical construction while preserving gameplay. Examples include rebuilding the kitchen suite, refrigerator, sink, sectional couch, dining set, beds, bath fixtures, stairs, doors, and character sprites without inventing unrelated layouts.

## Production requirement

The overhaul covers every playable floor, exterior area, character, dog, vehicle, furniture object, appliance, architectural kit, effect, activity pose, and interface surface. Work may be committed in auditable tested batches, but the final acceptance target is the reference quality level across the complete game. Temporary generic assets must remain honestly marked temporary until replaced and browser approved.
